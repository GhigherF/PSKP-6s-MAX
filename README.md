# Microblog — документация изменений

## Запуск

```
build.bat   — сборка образов и создание контейнеров
start.bat   — запуск контейнеров
stop.bat    — остановка контейнеров
```

При первом запуске обязательно `build.bat`, затем `start.bat`.
При изменении схемы БД: `docker compose down -v` затем `build.bat` и `start.bat`.

---

## Архитектура

| Сервис    | Описание                        |
|-----------|---------------------------------|
| postgres  | База данных PostgreSQL 16       |
| backend   | Node.js/Express + WebSocket     |
| frontend  | React SPA, собранный nginx      |
| nginx     | Reverse proxy, точка входа :80  |

Redis удалён — кэширование постов убрано, так как посты после просмотра уходят в архив и повторно не показываются.

## Функционал

### 1. Лайки и комментарии через WebSocket
- При открытии комментариев под постом фронтенд подключается к `/ws?postId=N`
- Все клиенты в одной "комнате" (один postId) получают события в реальном времени:
  - `type: "comment"` — новый комментарий
  - `type: "comment_deleted"` — удалённый комментарий
  - `type: "comment_like"` — обновление счётчика лайков комментария
  - `type: "like"` — обновление счётчика лайков поста
- Лайки постов и комментариев — toggle (повторное нажатие снимает лайк)

### 2. Просмотренные посты
**Как определяется просмотр:** пост считается просмотренным через 2 секунды после того как он появился на экране (загрузился в ленте). Это реализовано через `setTimeout` на фронтенде.

- ID просмотренных постов хранятся в `localStorage` (ключ `viewed_posts`) — работает без авторизации
- При авторизованном просмотре факт фиксируется в таблице `post_views` через `POST /posts/:id/view`
- При следующей загрузке ленты просмотренные посты исключаются через параметр `?exclude=1,2,3`
- В профиле пользователя вкладка "👁 Просмотренные" показывает список просмотренных постов в порядке убывания даты просмотра (только для владельца профиля)

### 3. Профиль пользователя
Вкладки:
- **📊 Статистика** — количество постов, комментариев, полученных лайков на постах и комментариях, просмотров постов
- **💬 Комментарии** — все комментарии пользователя с возможностью перейти к посту
- **👁 Просмотренные** — только для владельца, список просмотренных постов
- **✏️ Редактировать** — только для владельца, смена ника и аватарки

### 4. Удаление постов и комментариев
- Обычный пользователь может удалить свои посты и комментарии
- Администратор может удалить любой пост и любой комментарий
- Удалённый администратором комментарий отображается с пометкой "🚫 Удалено администратором"
- Удаление мягкое (поле `deleted = TRUE`), данные из БД не стираются

### 5. Подписки
- Кнопка "Подписаться/Отписаться" на странице профиля другого пользователя
- Посты подписанных пользователей отображаются первыми в ленте, отделены полоской "📌 Посты подписок"
- Ниже идут остальные посты с разделителем "🌐 Остальные посты"

### 6. Аватарки и смена ника
- В профиле на вкладке "✏️ Редактировать" можно загрузить аватарку (любой image/*)
- Там же можно сменить никнейм (проверяется уникальность)
- Аватарка отображается рядом с ником в постах, комментариях и профиле

## API эндпоинты

| Метод  | Путь                      | Доступ       | Описание                          |
|--------|---------------------------|--------------|-----------------------------------|
| POST   | /auth/register            | публичный    | Регистрация                       |
| POST   | /auth/login               | публичный    | Вход                              |
| GET    | /auth/me                  | авторизован  | Текущий пользователь              |
| GET    | /posts                    | публичный    | Лента постов (с exclude)          |
| GET    | /posts/:id                | публичный    | Один пост                         |
| POST   | /posts                    | авторизован  | Создать пост                      |
| DELETE | /posts/:id                | авторизован  | Удалить пост                      |
| POST   | /posts/:id/view           | авторизован  | Отметить просмотренным            |
| POST   | /posts/:id/like           | авторизован  | Toggle лайк поста                 |
| GET    | /posts/:id/comments       | публичный    | Комментарии поста                 |
| POST   | /posts/:id/comments       | авторизован  | Добавить комментарий              |
| DELETE | /comments/:id             | авторизован  | Удалить комментарий               |
| POST   | /comments/:id/like        | авторизован  | Toggle лайк комментария           |
| GET    | /users/:id                | публичный    | Профиль + статистика              |
| PATCH  | /users/me                 | авторизован  | Обновить ник/аватарку             |
| GET    | /users/:id/comments       | публичный    | Комментарии пользователя          |
| GET    | /users/:id/views          | владелец     | Просмотренные посты               |
| POST   | /users/:id/subscribe      | авторизован  | Toggle подписка                   |

## WebSocket
- URL: `ws://host/ws?postId=N`
- Сервер группирует клиентов по postId в комнаты
- События: `like`, `comment`, `comment_deleted`, `comment_like`
